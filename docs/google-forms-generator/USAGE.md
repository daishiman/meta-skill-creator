# 使い方ガイド - Google Forms Generator

Google Forms Generatorの詳細な使い方を説明します。

> **前提**: プラグインのインストールとOAuth認証設定が完了している必要があります。
> 未完了の場合は [インストールガイド](INSTALLATION.md) を先に参照してください。

---

## 概要

Google Forms Generatorは**対話形式のヒアリング**でフォームの要件を収集し、
Google Forms APIを使用してフォームを自動生成します。

```
フォーム作成を依頼
    ↓
Phase 1: ヒアリング（目的・質問項目・設定を収集）
    ↓
Phase 2: 構成設計（Markdownで確認）
    ↓
Phase 3: API実行（フォーム作成）
    ↓
Phase 4: 結果報告（URL報告 + ファイル保存）
```

---

## 基本的な使い方

### 方法1: 自然言語で話しかける

```
「Googleフォームを作成したい」
「顧客満足度アンケートを作って」
「イベント申込フォームが欲しい」
```

### 方法2: テンプレートから作成

```
「テンプレートからフォームを作りたい」
「アンケートテンプレートを使って」
```

---

## Phase 1: ヒアリング

ユーザーから以下の情報を収集します。

### 収集する情報

| 項目 | 説明 | 例 |
|------|------|-----|
| フォームの目的 | アンケート/申込/クイズ/問合せ等 | 顧客満足度調査 |
| タイトル | フォームのタイトル | 2026年顧客満足度アンケート |
| 説明文 | フォーム上部に表示する説明 | ご協力をお願いします |
| 質問項目 | 質問文・タイプ・選択肢・必須/任意 | 満足度を5段階で評価 |
| 設定オプション | クイズモード・メール収集等 | メールアドレスを収集 |
| 保存先 | Google Driveのフォルダ（オプション） | フォルダID指定 |

### 対話例

```
ユーザー: 「顧客満足度アンケートを作って」

Claude: フォームの詳細をヒアリングさせてください。
        まず、アンケートの対象者は誰ですか？
        1. 既存顧客
        2. 新規顧客
        3. 全顧客
        4. その他

ユーザー: 「1」

Claude: 質問項目を一つずつ確認します。
        最初の質問は何ですか？
        1. 満足度（5段階評価）
        2. サービスの改善点（自由記述）
        3. 利用頻度（選択式）
        4. カスタム（自分で指定）
```

---

## Phase 2: 構成設計

ヒアリング結果をもとに、フォーム構成をMarkdownで出力します。

### 確認用Markdown出力例

```markdown
# 顧客満足度アンケート - フォーム構成確認

## 基本情報
| 項目 | 値 |
|------|-----|
| タイトル | 顧客満足度アンケート |
| 説明 | サービス改善のためにご協力をお願いいたします |
| クイズモード | OFF |
| メール収集 | VERIFIED |

## 質問一覧
| # | 質問文 | タイプ | 必須 | 選択肢 |
|---|--------|--------|------|--------|
| 1 | 総合満足度 | SCALE | はい | 1-5 |
| 2 | 良かった点 | LONG_TEXT | いいえ | - |
| 3 | 改善してほしい点 | LONG_TEXT | いいえ | - |

## この構成でフォームを作成しますか？
```

ユーザーが承認すると、Phase 3に進みます。

---

## Phase 3: API実行

承認後、Google Forms APIでフォームを自動作成します。

### 実行される処理

1. フォーム作成（タイトルのみ）
2. 質問・設定追加（batchUpdate）
3. 公開設定
4. フォルダ移動（指定時）

---

## Phase 4: 結果報告

作成完了後、以下の情報を報告します。

### 報告内容

| 項目 | 内容 |
|------|------|
| タイトル | フォーム名 |
| 回答用URL | 回答者に共有するURL |
| 編集用URL | フォーム編集画面のURL |
| フォームID | APIで使用するID |
| 保存先 | Google Driveのフォルダ |

### ファイル保存

結果は `05_Project/GoogleFrom/` に自動保存されます：

```
05_Project/GoogleFrom/
└── {YYYYMMDD_HHMMSS}_{タイトル}/
    ├── 01-design.md   # 下書き・設計情報
    └── 02-result.md   # URL情報・スプレッドシート情報
```

## テンプレートの活用

6種類のテンプレートから素早くフォームを作成できます。

| テンプレート | 用途 |
|-------------|------|
| survey | 顧客満足度調査 |
| event-registration | イベント申込 |
| contact | お問い合わせフォーム |
| quiz | クイズ・テスト |
| feedback | フィードバック収集 |
| custom | 白紙（カスタム） |

```
「テンプレートからフォームを作りたい」
→ テンプレート一覧が表示されるので選択
```

---

## 既存フォームの修正

作成済みフォームの質問追加・削除・変更が可能です。

```
「フォームを修正したい」
「質問を追加して」
「質問の順番を変えたい」
```

フォームIDを指定して修正します。

## 回答データの取得

フォームの回答データを取得・エクスポートできます。

```
「フォームの回答を取得したい」
「回答をスプレッドシートにエクスポートして」
```

## 質問タイプ詳細

### テキスト系

| タイプ | 用途 |
|--------|------|
| `SHORT_TEXT` | 名前、メールアドレスなど短い回答 |
| `LONG_TEXT` | 自由記述、感想など長い回答 |

### 選択系

| タイプ | 用途 |
|--------|------|
| `RADIO` | 単一選択（ラジオボタン） |
| `CHECKBOX` | 複数選択 |
| `DROP_DOWN` | プルダウン選択 |

### スケール・評価系

| タイプ | 用途 |
|--------|------|
| `SCALE` | 線形スケール（例: 1〜5） |
| `RATING` | 星/ハートで評価 |

### グリッド系

| タイプ | 用途 |
|--------|------|
| `GRID_RADIO` | 行×列の単一選択グリッド |
| `GRID_CHECKBOX` | 行×列の複数選択グリッド |

### 日時系

| タイプ | 用途 |
|--------|------|
| `DATE` | 日付の入力 |
| `TIME` | 時刻の入力 |

## ベストプラクティス

### 1. 目的を明確に伝える

```
✅ 「社内イベントの参加者を募るフォームを作って。50名定員で、名前・部署・参加日を聞きたい」
❌ 「フォームを作って」
```

### 2. 質問項目を事前に整理する

質問文、回答形式、必須/任意をあらかじめ決めておくとスムーズです。

### 3. テンプレートを活用する

用途が定型的な場合、テンプレートから始めるのが最速です。

### 4. 構成確認は慎重に

Phase 2で表示される構成確認Markdownをしっかり確認してから承認してください。
